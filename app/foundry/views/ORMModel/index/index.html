Regenerating Application Model Class Files

Whenever changes are made to the application model file, it is necessary to re-create the PHP object class files to reflect the changes. When you regenerate, you may find that your schema (database tables) have become out of date. Any loss of synchronization will be displayed here, along with options for automatically resolving the difference.

Model-to-Database Synchronization

Foundry can compare the differences (if any) between your application model and the default data source specified in your application's connections.config.php file. The analysis results will include options for rectifying any differences found.
